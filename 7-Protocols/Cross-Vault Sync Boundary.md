# Cross-Vault Sync Boundary

## Rule

如果未来要在多个 vault 之间做同步，只同步 Xavier 本人的长期认知，不同步项目和执行上下文。

## 允许同步的内容

- 身份定位
- 偏好
- 价值排序
- 目标
- 反复出现的模式
- strengths
- blindspots
- commitments
- 高层 current state

这些内容的共同特征是：
- 主要关于 Xavier 这个人
- 具有跨系统长期价值
- 不属于某个单独项目的上下文

## 不允许同步的内容

- 项目进展
- 仓库级任务状态
- 某次实现的技术细节
- 特定项目的决策记录
- 一次性执行过程
- 低价值闲聊内容

## 原则

1. `codex` 可以作为主参考源，但不能把四个 vault 机械压成完全一样。
2. 同步 Xavier 相关内容时，要保留每个系统自己的角色页、总纲页和工作流页。
3. 冲突信息要标出来，不要静默覆盖。
4. 短期波动先不要同步成长期认知。
