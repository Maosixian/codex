# Memory Write Policy

## Rule

不是所有信息都该进入长期系统。

## 应该写入的内容

- Xavier 的稳定偏好、价值、目标、模式
- 项目的长期事实、关键决策、长期风险
- 可复用的方法和流程
- 对未来恢复工作明显有帮助的 session 结论

## 不该轻易写入的内容

- 一时情绪波动
- 一次性闲聊
- 未经验证的猜测
- 没有长期价值的细枝末节

## 写入要求

- 先判断属于哪一层
- 有旧页就优先更新旧页
- 需要时补 cross-link
- 不要让对话成为唯一来源

## Authorization Boundary

- 对记录、补记、更新 Obsidian 中的长期记忆和工作记录，默认视为 Codex 的常规职责，不需要逐次单独授权。
- 只要内容符合本页的写入标准，Codex 应主动写回，而不是等待 Xavier 提醒。
- 删除 vault 内容、清空页面或做不可逆移除前，必须先得到 Xavier 明确授权。
