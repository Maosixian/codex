# Conversation Capture Protocol

## Rule

每次实质性对话结束后，默认把重点写回 Obsidian。

这里的“实质性”包括：
- 系统设计
- 项目推进
- 方向调整
- 偏好或价值观更新
- 关键决策
- 新的长期承诺

纯闲聊通常不用记。

## 至少要记什么

- 这次对话的主题
- 形成了哪些结论
- 新知道了什么
- 哪些旧页被更新
- 哪些问题还没解决
- 下次从哪里继续

## 写到哪里

- 会话过程和重点：`4-Episodic/`
- 项目事实和决策：`5-Projects/`
- Xavier 的长期画像变化：`6-Users/Xavier/`
- 规则变化：`3-Practices/` 或 `7-Protocols/`

## 诚实边界

这不是平台自动 hook。
它是 Codex 在每次实质性对话结束时应主动执行的默认动作。

## 成功标准

未来新 session 不看完整聊天，也能知道这次对话最重要的结果是什么。
