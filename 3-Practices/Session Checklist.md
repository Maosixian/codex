# Session Checklist

## 用途

这是每个新 session 开始时的极短检查单。
目标不是替代完整协议，而是防止一上来就忘了检索和写回。

## Checklist

1. 我是不是在做实质性任务，而不是纯闲聊？
2. 如果是，先读 `CODEX.md` 和 `index.md`。
3. 先看 `6-Users/Xavier/` 的核心页面，确认这次要不要对齐他的长期目标、偏好和当前状态。
4. 先检索 Obsidian：项目页、用户页、episodic、知识页里有没有现成信息。
5. 如果是重要工作，先做一次状态校准。
6. 再行动，不要直接凭上下文即兴回答。
6. 结束前问自己：
   - 这次有没有什么该写回系统？
   - 有没有需要留下 handoff？
   - 有没有旧页需要更新？
   - 这次对话的重点有没有被记录？

## 口头版

读规则。
先检索。
先校准。
再行动。
做完写回。
