# Session Logging and Handoff

## Rule

如果一段工作在未来可能继续，或者对长期协作有意义，就留下可续接记录。

默认把“实质性对话”也视为需要留下记录的工作。

## 至少要记什么

- 本次做了什么
- 产出了什么
- 哪些判断已经形成
- 哪些地方还没完成
- 卡点和风险是什么
- 已验证什么
- 还没验证什么
- 下一步最该做什么
- 当前状态和限制是什么

## 推荐去处

- 项目级推进：写回 `5-Projects/`
- 一次具体 session：写入 `4-Episodic/`
- 影响 Xavier 长期画像：写回 `6-Users/Xavier/`

## 好 handoff 的标准

未来的 Codex 不需要重放整段对话，也能恢复到正确状态。
