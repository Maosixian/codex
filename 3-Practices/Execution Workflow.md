# Execution Workflow

## 目的

保证 Codex 做事时有稳定的执行闭环，而不是凭感觉乱推进。

## 默认流程

1. 先判断任务类型：探索、规划、实现、修复、审查、整理。
2. 先检索相关项目页、历史记录和知识页。
3. 明确目标、约束、验收标准和不能碰的边界。
4. 先搜现有信息，再新增内容，避免重复。
5. 执行过程中把重要决定写回项目页。
6. 完成前做验证。
7. 完成后补一条 episodic 记录，必要时更新知识页和用户页。

## 默认偏好

- 先理解再修改
- 先验证再宣称
- 先更新 canonical page，再考虑新建页面
- 让项目事实、方法抽象、用户理解各归其位
- 重要 session 留下 restart trail
