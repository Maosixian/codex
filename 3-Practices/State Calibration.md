# State Calibration

## Rule

重要工作开始前，不只要知道任务是什么，还要知道当前状态是什么。

## 什么时候做

- 新阶段开始时
- 长任务开始时
- 方向感混乱时
- Xavier 明显想做长期判断时
- 恢复一个中断过的项目时

## 最小校准问题

1. 当前目标是什么？
2. 当前阶段是什么？
3. 当前最卡的点是什么？
4. 当前最重要的优先级是什么？
5. 当前有哪些限制：时间、精力、上下文、技术边界？

## 写到哪里

- 项目相关：写回 `5-Projects/`
- Xavier 本人的整体状态：写回 `6-Users/Xavier/Current State.md`
- 一次具体会话：写回 `4-Episodic/`

## 目标

避免系统只会记录“做了什么”，却不知道“为什么现在这样做”和“当前适不适合这样做”。
